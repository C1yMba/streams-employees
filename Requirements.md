# Требования к системе Streams Employees

## User Stories

### US-001: Просмотр статистики по департаменту
**Актор:** HR-менеджер
**Описание:** Как HR-менеджер, я хочу получать статистику по зарплатам в конкретном департаменте

**Критерии приемки:**
- GET /department/{id}/salary/sum - возвращает сумму всех зарплат в департаменте
- GET /department/{id}/salary/min - возвращает сотрудника с минимальной зарплатой
- GET /department/{id}/salary/max - возвращает сотрудника с максимальной зарплатой
- Время ответа не превышает 1 секунду

### US-002: Управление списком сотрудников
**Актор:** HR-менеджер
**Описание:** Как HR-менеджер, я хочу управлять списком сотрудников в системе

**Критерии приемки:**
- Просмотр всех сотрудников по департаментам
- Добавление нового сотрудника
- Удаление сотрудника по ID
- Поиск сотрудника по ID

### US-003: Получение рекомендаций
**Актор:** HR-менеджер  
**Описание:** Как HR-менеджер, я хочу получать автоматические рекомендации по работе с сотрудниками

**Критерии приемки:**
- GET /api/recommendations/employee/{id} - возвращает рекомендации для сотрудника
- Рекомендации генерируются на основе статических правил
- Поддерживаются типы: повышение зарплаты, обучение, продвижение

### US-004: Управление кешем системы
**Актор:** Системный администратор
**Описание:** Как системный администратор, я хочу управлять кешем для оптимизации производительности

**Критерии приемки:**
- POST /api/cache/clear - очистка кеша
- GET /actuator/health - проверка состояния приложения
- Кеширование списка сотрудников

## Нефункциональные требования

### NFR-001: Производительность
- Время отклика API: не более 1 секунды для операций чтения
- Время отклика API: не более 2 секунд для операций записи
- Поддержка до 100 одновременных пользователей

### NFR-002: Надежность  
- Graceful обработка ошибок с понятными сообщениями
- Валидация входных данных
- Логирование операций

### NFR-003: Удобство использования
- Автоматическая генерация документации API
- Понятные названия эндпоинтов
- Консистентная структура ответов API

## Статические правила рекомендаций

### Правило 1: Рекомендация повышения зарплаты
Если зарплата сотрудника меньше средней по департаменту на 20% или больше, 
генерируется рекомендация о повышении зарплаты.

### Правило 2: Рекомендация обучения
Для всех сотрудников с зарплатой ниже медианной в департаменте
генерируется рекомендация о дополнительном обучении.

### Правило 3: Рекомендация продвижения
Сотрудники с зарплатой в топ-10% департамента получают 
рекомендацию о возможном продвижении.
